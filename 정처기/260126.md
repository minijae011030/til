## 🔹 객체지향 / 설계

- 인터페이스 분리 원칙(ISP)
  → 인터페이스를 작게 나눠서 필요한 것만 사용
- 상위 설계 (High-Level)
  → 아키텍처 / 인터페이스 / UI
- 하위 설계 (Low-Level)
  → 모듈 설계
- 결합도
  - 데이터 결합: 값만 전달
  - 스탬프 결합: 배열, 객체 같은 자료구조 전달
- BCNF
  → 결정자가 무조건 후보키여야 함

## 🔹 테스트 / 품질 / 애자일

- 알파 테스트
  → 사용자 앞에서 개발자가 같이 테스트
- 테스트 오라클
  → 이게 맞는 결과인지 판단 기준
- 블랙박스 테스트
  → 동치 분해 / 경계값 / 원인결과 그래프
- 화이트박스 테스트
  → 기초 경로 검사
- 정적 분석 도구
  → pmd / checkstyle / cppcheck
- 애자일 아님
  → 모듈 중심 개발

## 🔹 DB / SQL / 정규화

- CRUD 분석
  → 프로세스 vs 테이블 매트릭스
- ALTER
  → 필드 추가, 테이블 구조 변경
- 슈퍼키
  → 유일성 O, 최소성 X
- 튜플 최대 개수
  → 도메인 값 전부 곱하기
- 로킹 단위
  → 작을수록 병행성 ↑
- 데이터 모델 요소
  → 구조 / 연산 / 제약

## 🔹 OS / 자료구조 / 알고리즘

- 폴딩법
  → 나눈 값 더하거나 XOR
- 세그먼트 주소 계산
  → 시작주소 + 변위값
- 커널 아님
  → 사용자 인터페이스 = 쉘
- until who | grep wow
  → wow 로그인할 때까지 반복

## 🔹 네트워크 / 기타

- ARP
  → IP(논리) → MAC(물리)
- OLAP 연산
  → Roll-up / Drill-down / Slice / Dice / Pivot
- SDDC
  → 하드웨어 상관없이 전부 소프트웨어로 관리 (가상화)
- SPICE
  → 소프트웨어 프로세스 잘 하는지 평가하는 국제 표준
- SADT
  → SoftTech, 블록 다이어그램 요구분석 도구
- WDM
  → 파장(Wavelength) 나눠서 여러 신호 보냄

## 🔹 언어

- Java long = 8byte
- PHP 연산자 아님 = #
- Python a[:7:2]
  → 0번부터 6번까지 2칸씩: [0,20,40,60]
