## HTML, CSS 면접질문

### ⭐ px, em, rem 의 차이, 또 반응형 웹을 구현 시 사용하는 단위

- px
  고정 단위로, 항상 동일한 크기를 가짐
  → 정밀한 디자인에는 좋지만 반응형에는 불리
- em
  부모 요소의 font-size를 기준으로 계산
  → 중첩이 깊어질수록 예측이 어려움
- rem
  최상위 요소(html)의 font-size를 기준
  → 전체 스케일 조정이 쉬워 반응형에 적합

반응형 웹에서는 rem을 주로 사용
폰트, 여백 등을 비율 기반으로 관리하기 좋기 때문

### ⭐ vw, vh

- vw (viewport width): 뷰포트 너비의 1%

- vh (viewport height): 뷰포트 높이의 1%

→ 화면 크기에 직접 반응하는 단위로, 풀스크린 레이아웃이나 히어로 섹션에 자주 사용

### ⭐ CSS 선택자의 우선순위

우선순위는 다음 기준으로 결정된다.

1. !important
2. 인라인 스타일
3. ID 선택자
4. 클래스 / 속성 / 가상 클래스
5. 태그 선택자
6. 동일 우선순위면 나중에 선언된 스타일

→ 구체성이 높을수록 우선순위가 높음

### ⭐ SCSS

SCSS는 CSS 전처리기로 다음 기능을 제공한다.

- 변수
- 중첩(Nesting)
- 믹스인
- 조건문, 반복문

→ CSS를 구조적으로 작성할 수 있고
→ 유지보수성과 재사용성이 향상됨

### ⭐ position

- static: 기본값, 위치 지정 불가
- relative: 자기 자신 기준 이동
- absolute: 가장 가까운 position 부모 기준
- fixed: 뷰포트 기준, 스크롤과 무관
- sticky: 스크롤 위치에 따라 relative ↔ fixed

### ⭐ margin과 padding의 차이

- margin: 요소 바깥 여백
- padding: 요소 내부 여백 (콘텐츠와 테두리 사이)

→ margin은 요소 간 거리, padding은 요소 내부 공간

### ⭐ HTML 렌더링 도중 JavaScript가 실행되면 렌더링이 멈추는 이유

HTML 파서는 JavaScript 실행 중 DOM 변경 가능성 때문에
JS를 만나면 파싱을 중단하고 스크립트를 먼저 실행함

→ 이를 렌더링 블로킹이라고 함

### ⭐ display 속성

- block
- inline
- inline-block
- none
- flex
- grid

→ 요소의 레이아웃 방식과 흐름을 결정

### ⭐ CSS 애니메이션과 JS 애니메이션의 차이

- CSS 애니메이션
  - 선언적
  - GPU 가속
  - 성능 좋음
  - 단순한 애니메이션에 적합
- JS 애니메이션
  - 제어가 자유로움
  - 복잡한 상호작용 가능
  - 성능 관리 필요

### ⭐ CSS in JS의 장단점

장점

- 컴포넌트 단위 스타일 관리
- 스타일 충돌 방지
- 동적 스타일링 용이

단점

- 런타임 오버헤드
- 번들 크기 증가 가능
- 러닝 커브 존재

### ⭐ 시맨틱 마크업

의미 있는 HTML 태그를 사용해 구조를 표현하는 것
예: header, nav, main, section, article

→ 접근성, SEO, 유지보수성 향상

### ⭐ HTML5의 태그

- 구조: header, footer, section, article
- 미디어: video, audio
- 입력: input, datalist, progress

### ⭐ script 태그에서 Async와 Defer의 차이

- async
  - 다운로드 즉시 실행
  - 실행 순서 보장 ❌
- defer
  - HTML 파싱 완료 후 실행
  - 실행 순서 보장 ⭕

→ 일반적으로 defer를 더 많이 사용

### ⭐ 가상 클래스

요소의 상태를 기준으로 스타일 적용

예: :hover, :focus, :active, :nth-child

### ⭐ flex와 grid 차이

flex는 1차원적 레이아웃 시스템으로 요소를 가로 또는 세로 방향으로 배치하는데 용이하고, grid는 2차원 레이아웃 시스템으로 행과 열을 모두 고려한 레이아웃 배치가 가능합니다.

- flex
  - 1차원 레이아웃
  - 한 방향 정렬에 강점
- grid
  - 2차원 레이아웃
  - 복잡한 페이지 구조에 적합

### ⭐ z-index

요소 쌓이는 순서를 결정하는 것을 말하고 숫자가 클수록 상위로 갑니다. position relative, absolute, fixed, sticky에서만 사용가능합니다.

요소의 쌓임 순서를 제어

숫자가 클수록 위에 표시됨

⚠️ 단, stacking context 내부에서만 비교 가능
position이 있고 z-index가 지정된 요소가 기준이 됨

### ⭐ Tailwind CSS 설명

유틸리티 클래스 기반 CSS 프레임워크
미리 정의된 작은 클래스들을 조합해 스타일링

장점

- 빠른 개발
- 일관된 디자인
- 별도 CSS 작성 최소화

단점

- 클래스가 길어질 수 있음
- 초반 가독성 부담

### ⭐ Tailwind CSS 반응형 구현 방식

반응형 프리픽스 사용:

- sm:
- md:
- lg:
- xl:

→ 미디어 쿼리 없이 뷰포트 기준 스타일 제어 가능
