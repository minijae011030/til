# TypeScript

## 1. TypeScript 개요

### ⭐ 타입스크립트를 사용하는 이유

타입스크립트는 자바스크립트에 정적 타입을 추가한 언어입니다. 1. 컴파일 단계에서 오류를 미리 발견

- 런타임 에러 감소
- 코드 안정성 향상 2. 코드 가독성과 유지보수성 향상
- 변수, 함수의 역할이 명확
- 협업 시 의도 파악 용이 3. 개발 생산성 향상
- 자동 완성, 타입 추론
- 안전한 리팩토링 가능

### ⭐ 타입스크립트가 컴파일 언어인 이유

브라우저는 TypeScript를 직접 이해하지 못하므로
컴파일을 통해 JavaScript로 변환됩니다.

- 타입 정보는 컴파일 시점에만 존재
- 런타임에는 제거됨

→ 타입은 개발자를 위한 안전장치

## 2. 타입 정의 방식

### ⭐ Type vs Interface 차이

둘 다 타입을 정의하지만 목적과 확장 방식에 차이가 있습니다.

- Interface

  - 객체 구조 정의에 특화
  - extends로 확장 가능
  - 선언 병합 가능

- Type

  - 객체, 유니온, 튜플 등 다양한 타입 표현 가능
  - 선언 병합 불가

→ 객체 구조 중심이면 Interface
→ 복합 타입 표현이면 Type

### ⭐ Literal Type

특정 값만 허용하도록 타입을 제한합니다.

```ts
type Direction = "left" | "right";
```

→ 상태값, 옵션, 액션 타입에 자주 사용
→ 오타로 인한 버그 방지

### ⭐ Enum vs Union Literal

#### Enum

```ts
enum Status {
  Ready,
  Loading,
  Done,
}
```

#### Union Literal

```ts
type Status = "ready" | "loading" | "done";
```

- Enum: 런타임 코드 생성
- Union Literal: 런타임 코드 없음 → 번들 크기 유리

→ 최근에는 Union Literal 선호

## 3. 타입 시스템 핵심 개념

### ⭐ 타입 추론 (Type Inference)

명시적 타입 선언 없이도
코드를 기반으로 타입을 자동 추론합니다.

```ts
let count = 0; // number로 추론
```

→ 불필요한 타입 선언 감소
→ 가독성과 생산성 향상

### ⭐ Union Type & Narrowing

Union Type은 여러 타입 중 하나를 허용합니다.

```ts
let value: string | number;
```

조건문을 통해 타입을 좁혀 인식합니다.

```ts
if (typeof value === "string") {
  value.toUpperCase();
}
```

→ 런타임 에러를 컴파일 단계에서 방지

### ⭐ never 타입

절대 발생하지 않는 값을 표현합니다.

```ts
function throwError(): never {
  throw new Error();
}
```

- exhaustive check에 사용
- switch 문 안정성 강화

## 4. 제네릭과 타입 재사용

### ⭐ 제네릭 (Generic)

타입을 매개변수처럼 받아 사용하는 문법입니다.

- 사용 시점에 타입 결정
- 재사용성과 타입 안정성 확보

### ⭐ 제네릭 유틸리티 타입

기존 타입을 변형해 새로운 타입을 생성합니다.

- Partial : 모든 속성 optional
- Required : 모든 속성 필수
- Pick<T, K> : 특정 속성 선택
- Omit<T, K> : 특정 속성 제외
- Readonly : 읽기 전용

## 5. 객체와 클래스 관련 타입

### ⭐ readonly

객체의 속성을 변경 불가능하게 만듭니다.

```ts
interface User {
  readonly id: number;
}
```

→ 불변성 유지
→ React state, props 타입 정의에 자주 사용

### ⭐ Optional Property (?)

속성이 있을 수도, 없을 수도 있음을 표현합니다.

```ts
interface User {
  name: string;
  age?: number;
}
```

→ API 응답 타입
→ 부분 업데이트에 유용

### ⭐ 클래스 접근 제어자

클래스 멤버의 접근 범위를 제어합니다.

- public
  - 기본값
  - 어디서든 접근 가능
- private
  - 클래스 내부에서만 접근 가능
- protected
  - 클래스 내부 + 상속받은 클래스

→ 캡슐화와 안정적인 객체 설계

## 6. 타입 안전성 관련 개념

### ⭐ 타입 단언 (Type Assertion)

개발자가 타입을 더 잘 알고 있을 때
컴파일러에 타입을 강제로 알려줍니다.

```ts
const el = document.getElementById("root") as HTMLElement;
```

⚠️ 잘못 사용하면 타입 안정성 저하
→ 정말 필요한 경우에만 사용

### ⭐ any vs unknown

- any
  - 타입 검사 완전 무시
  - 매우 위험
- unknown
  - 타입은 알 수 없음
  - 사용 전 타입 체크 강제

→ unknown이 더 안전한 선택
