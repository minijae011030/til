## 타입스크립트

### ⭐ 타입스크립트를 사용하는 이유에 대해 설명해주세요.

타입스크립트는 자바스크립트에 정적 타입을 추가한 언어

1. 컴파일 단계에서 오류를 미리 발견

- 런타임 에러 감소
- 코드 안정성 향상

2. 코드 가독성과 유지보수성 향상

- 변수, 함수의 역할이 명확해짐
- 협업 시 의도 파악이 쉬움

3. 개발 생산성 향상

- 자동 완성, 타입 추론
- 리팩토링이 안전해짐

### ⭐ 타입스크립트는 왜 컴파일 언어인가요?

타입스크립트는 브라우저가 이해하지 못하므로
컴파일을 통해 자바스크립트로 변환됩니다.

- 타입 정보는 컴파일 시점에만 존재
- 런타임에는 제거됨

→ “타입은 개발자를 위한 안전장치”

### ⭐ Type, Interface의 차이

둘 다 타입을 정의하는 데 사용되지만 목적과 확장 방식에 차이가 있습니다.

Interface

- 객체 구조 정의에 특화
- extends로 확장 가능
- 선언 병합 가능

Type

- 객체, 유니온, 튜플 등 더 다양한 타입 표현 가능
- 선언 병합 불가

### ⭐ 제네릭

제네릭은 타입을 매개변수처럼 받아 사용하는 문법

타입을 고정하지 않고 사용 시점에 결정할 수 있어
재사용성과 타입 안정성을 동시에 확보 가능

### ⭐ 제네릭 유틸리티 타입

기존 타입을 변형해 새로운 타입을 만들어주는 타입 도구

- Partial<T> : 모든 속성을 optional
- Required<T> : 모든 속성을 필수
- Pick<T, K> : 특정 속성만 선택
- Omit<T, K> : 특정 속성 제외
- Readonly<T> : 읽기 전용

### ⭐ 클래스 접근 제어자 (Public / Private / Protected)

클래스 멤버의 접근 범위를 제어합니다.

- public
  - 기본값
  - 어디서든 접근 가능
- private
  - 클래스 내부에서만 접근 가능
- protected
  - 클래스 내부 + 상속받은 클래스에서 접근 가능

→ 캡슐화와 안정적인 객체 설계를 위해 사용

### ⭐ 타입 추론 (Type Inference)

타입스크립트는 명시적으로 타입을 작성하지 않아도
코드를 기반으로 타입을 자동으로 추론합니다.

```ts
let count = 0; // number로 추론
```

→ 불필요한 타입 선언을 줄이고,
→ 코드 가독성과 생산성을 높여줍니다.

### ⭐ Union Type & Narrowing

Union Type은 여러 타입 중 하나가 될 수 있음을 의미합니다.

```ts
let value: string | number;
```

타입스크립트는 조건문을 통해 타입을 자동으로 좁혀(Narrowing) 인식합니다.

```ts
if (typeof value === "string") {
  value.toUpperCase();
}
```

→ 런타임 에러를 컴파일 단계에서 방지

### ⭐ 타입 단언 (Type Assertion)

개발자가 타입을 더 잘 알고 있을 때
컴파일러에게 타입을 강제로 알려주는 문법입니다.

```ts
const el = document.getElementById("root") as HTMLElement;
```

⚠️ 잘못 사용하면 타입 안정성을 해칠 수 있어
정말 필요한 경우에만 사용

### ⭐ Literal Type

특정 값만 허용하도록 타입을 제한할 수 있습니다.

```ts
type Direction = "left" | "right";
```

→ 상태값, 옵션 값, 액션 타입 등에 자주 사용
→ 오타로 인한 버그 방지

### ⭐ Enum vs Union Literal

Enum은 관련된 상수 집합을 정의할 때 사용합니다.

```ts
enum Status {
  Ready,
  Loading,
  Done,
}
```

하지만 최근에는 번들 크기와 단순함 때문에
Union Literal을 더 많이 사용합니다.
(Union Literal은 런타임 코드가 생성되지 않아 번들 크기에 유리)

```ts
type Status = "ready" | "loading" | "done";
```

### ⭐ readonly

객체의 속성을 변경 불가능하게 만듭니다.

```ts
interface User {
  readonly id: number;
}
```

→ 불변성 유지
→ React state, props 타입 정의 시 자주 사용

### ⭐ Optional Property (?)

속성이 있을 수도, 없을 수도 있음을 표현합니다.

```ts
interface User {
  name: string;
  age?: number;
}
```

→ API 응답 타입, 부분 업데이트에서 유용

### ⭐ never 타입

절대 발생하지 않는 값을 표현하는 타입입니다.

```ts
function throwError(): never {
  throw new Error();
}
```

→ exhaustive check (모든 케이스 처리 확인)에 사용
→ switch 문 안정성 강화

### ⭐ any vs unknown

- any
  - 타입 검사 완전히 무시
  - 위험함
- unknown
  - 타입은 알 수 없지만
  - 사용 전 타입 체크 강제
