# HTML, CSS 면접 질문 리스트

## 1. HTML

- ⭐️ HTML이란 무엇인가요?
- ⭐️ 시맨틱 마크업이란 무엇이고, 왜 중요한가요?
- ⭐️ 시맨틱 태그를 사용하면 어떤 장점이 있나요?
- ⭐️ div 대신 section, article을 써야 하는 이유는 무엇인가요?
- meta 태그는 어떤 역할을 하나요?
- ⭐️ viewport meta 태그는 왜 필요한가요?
- ⭐️ HTML 파싱 도중 JavaScript가 실행되면 왜 렌더링이 멈추나요?
- ⭐️ script 태그에서 async와 defer의 차이는 무엇인가요?
- async와 defer 중 어떤 것을 더 많이 사용하고, 이유는 무엇인가요?

## 2. CSS

- ⭐️ CSS의 역할은 무엇인가요?
- ⭐️ block, inline, inline-block의 차이는 무엇인가요?
- display: none과 visibility: hidden의 차이는 무엇인가요?
- ⭐️ position 속성들의 차이를 설명해주세요.
- ⭐️ absolute 요소의 기준은 어떻게 결정되나요?
- fixed와 sticky의 차이는 무엇인가요?
- ⭐️ margin과 padding의 차이는 무엇인가요?
- ⭐️ CSS 선택자 우선순위는 어떻게 결정되나요?
- z-index는 언제 동작하지 않나요?
- ⭐️ stacking context란 무엇인가요?
- CSS 애니메이션과 JS 애니메이션의 차이는 무엇인가요?
- ⭐️ 성능 관점에서 CSS 애니메이션이 유리한 이유는 무엇인가요?

## 3. 반응형 웹

- ⭐️ 반응형 웹이란 무엇인가요?
- ⭐️ px, em, rem의 차이는 무엇인가요?
- ⭐️ rem이 반응형에서 선호되는 이유는 무엇인가요?
- vw, vh 단위는 어떤 경우에 사용하나요?
- ⭐️ flex와 grid 중 반응형 레이아웃에 더 적합한 것은 무엇인가요?
- ⭐️ 크로스 브라우징이란 무엇인가요?
- 크로스 브라우징을 위해 어떤 방법을 사용하나요?

## 4. Tailwind CSS

- ⭐️ Tailwind CSS란 무엇인가요?
- ⭐️ Tailwind CSS의 장점과 단점은 무엇인가요?
- Tailwind CSS를 사용하면 유지보수가 쉬워지는 이유는 무엇인가요?
- 클래스가 길어지는 문제는 어떻게 관리하나요?

- sm, md, lg 같은 프리픽스는 내부적으로 무엇을 의미하나요?
- Tailwind CSS는 언제 쓰는 게 적합하고, 언제 부적합한가요?
