# HTML, CSS

## 1. HTML (구조와 의미)

### ⭐ HTML이란

HTML은 웹 페이지의 구조와 의미를 정의하는 마크업 언어입니다.
콘텐츠의 역할과 관계를 표현하는 것이 목적입니다.

### ⭐ 시맨틱 마크업

의미 있는 태그로 문서 구조를 표현하는 것

예:

- header
- nav
- main
- section
- article
- footer

→ 접근성, SEO, 유지보수성 향상

### ⭐ HTML5 주요 태그

- 구조: header, footer, section, article
- 미디어: video, audio
- 입력: input, datalist, progress

### ⭐ meta 태그

문서의 메타데이터를 정의

- charset
- viewport
- description
- robots

→ SEO, 반응형, 브라우저 동작에 영향

### ⭐ HTML 렌더링 중 JavaScript 실행 시 멈추는 이유

HTML 파서는 JavaScript 실행 중 DOM 변경 가능성 때문에
script 태그를 만나면 파싱을 중단하고 JS를 먼저 실행합니다.

→ 렌더링 블로킹 현상

### ⭐ script 태그의 async와 defer 차이

- async
  - 다운로드 즉시 실행
  - 실행 순서 보장 ❌
- defer
  - HTML 파싱 완료 후 실행
  - 실행 순서 보장 ⭕

→ 일반적으로 defer 사용 권장

## 2. CSS (스타일과 레이아웃)

### ⭐ CSS 역할

CSS는 HTML로 정의된 구조에
레이아웃, 색상, 크기, 애니메이션 등 시각적 표현을 담당합니다.

### ⭐ display 속성

- block
- inline
- inline-block
- none
- flex
- grid

→ 요소의 배치 방식과 흐름 결정

### ⭐ position

- static: 기본값
- relative: 자기 자신 기준 이동
- absolute: 가장 가까운 position 부모 기준
- fixed: 뷰포트 기준
- sticky: 스크롤 위치에 따라 relative ↔ fixed

### ⭐ margin vs padding

- margin: 요소 바깥 여백
- padding: 요소 내부 여백

→ margin은 요소 간 거리
→ padding은 콘텐츠 내부 공간

### ⭐ CSS 선택자 우선순위

1. !important
2. 인라인 스타일
3. ID 선택자
4. 클래스 / 속성 / 가상 클래스
5. 태그 선택자
6. 동일 우선순위면 나중에 선언된 스타일

→ 구체성이 높을수록 우선

### ⭐ z-index와 stacking context

- z-index: 요소 쌓임 순서 제어
- position이 있어야 동작

stacking context 생성 조건:

- position + z-index
- opacity < 1
- transform 적용

→ 서로 다른 stacking context 간 z-index 비교 불가

### ⭐ CSS 애니메이션 vs JS 애니메이션

- CSS 애니메이션
  - 선언적
  - GPU 가속
  - 성능 좋음
  - 단순 애니메이션에 적합
- JS 애니메이션
  - 제어 자유로움
  - 복잡한 상호작용 가능
  - 성능 관리 필요

## 3. 반응형 웹 (Responsive Web)

### ⭐ px, em, rem 차이

- px
  - 고정 단위
  - 반응형에 불리
- em
  - 부모 font-size 기준
  - 중첩 시 계산 복잡
- rem
  - html font-size 기준
  - 전체 스케일 조정 쉬움

→ 반응형에서는 rem 사용 권장

### ⭐ vw, vh

- vw: 뷰포트 너비의 1%
- vh: 뷰포트 높이의 1%

→ 풀스크린, 히어로 섹션에 적합

### ⭐ flex와 grid 차이

- flex
  - 1차원 레이아웃
  - 정렬에 강점
- grid
  - 2차원 레이아웃
  - 복잡한 구조에 적합

→ 정렬은 flex, 구조는 grid

### ⭐ 크로스 브라우징

브라우저마다 다른 렌더링 차이를 고려해
모든 주요 브라우저에서 동일한 UX 제공

- CSS Reset
- Polyfill 사용

## 4. Tailwind CSS (유틸리티 기반 스타일링)

### ⭐ Tailwind CSS란

유틸리티 클래스 기반 CSS 프레임워크
미리 정의된 작은 클래스들을 조합해 스타일링

### ⭐ Tailwind CSS 장단점

장점

- 빠른 개발
- 일관된 디자인
- CSS 작성 최소화

단점

- 클래스가 길어질 수 있음
- 초반 가독성 부담

### ⭐ Tailwind CSS 반응형 구현 방식

반응형 프리픽스 사용

- sm:
- md:
- lg:
- xl:

→ 미디어 쿼리 없이 뷰포트 기준 스타일 제어 가능
