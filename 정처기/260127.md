## ✅ UML / 객체지향 / 설계

### Sequence Diagram (순차 다이어그램)

- 동적 모델링
- 시간 흐름 표현 (세로 ↓)
- 객체 간 메시지 흐름

❌ 정적 모델링 아님

### Use Case 관계 4가지

| 관계                    | 의미                   |
| ----------------------- | ---------------------- |
| 연관 (Association)      | 액터 ↔ 유스케이스 연결 |
| 포함 (Include)          | 항상 수행              |
| 확장 (Extend)           | 조건부                 |
| 일반화 (Generalization) | 상속 관계              |

### Encapsulation (캡슐화)

속성과 연산을 하나로 묶는 것

### MVC

| 구성       | 역할        |
| ---------- | ----------- |
| Model      | 데이터/로직 |
| View       | 화면        |
| Controller | Model 제어  |

❌ Model이 전달자 역할 아님

### XP (익스트림 프로그래밍)

- 애자일
- 문서보다 코드
- 소규모 / 변경 많은 환경

❌ 구조적 방법론 아님

## ✅ 테스트 / 품질

### Walkthrough vs Inspection

|      | Walkthrough   | Inspection |
| ---- | ------------- | ---------- |
| 주도 | 개발자        | 전문가     |
| 방식 | 설명하며 검토 | 공식 검토  |

### Stub vs Driver

|        | 역할                  |
| ------ | --------------------- |
| Stub   | 호출 당하는 가짜 모듈 |
| Driver | 호출 하는 가짜 모듈   |

### 단위 테스트 도구

- JUnit
- CppUnit
- HttpUnit
- NUnit

## ✅ DB / 정규화 / 트랜잭션

### BCNF

결정자가 후보키

### Candidate Key

유일성 + 최소성

### DCL

권한 관리 (GRANT, REVOKE)

### 트랜잭션 상태

| 상태                | 의미                       |
| ------------------- | -------------------------- |
| Active              | 실행 중                    |
| Partially Committed | 마지막 연산 끝, DB 반영 전 |
| Committed           | 반영 완료                  |
| Aborted             | 취소                       |
