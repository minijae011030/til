# TypeScript 면접 질문 리스트

## 1. TypeScript 개요

- TypeScript는 무엇이며 JavaScript와 어떤 관계인가요?
- TypeScript를 사용하는 이유는 무엇인가요?
- 정적 타입이 주는 장점은 무엇인가요?
- 런타임 에러와 컴파일 타임 에러의 차이는 무엇인가요?
- TypeScript는 왜 컴파일 언어라고 하나요?
- TypeScript 컴파일 후 타입 정보는 어떻게 되나요?

## 2. type vs interface

- type과 interface의 차이는 무엇인가요?
- interface는 언제, type은 언제 사용하는 게 좋나요?
- interface의 선언 병합이란 무엇인가요?
- enum과 union literal의 차이는 무엇인가요?
- enum이 런타임 코드를 생성한다는 말은 무슨 의미인가요?
- 왜 최근에는 union literal을 더 선호하나요?

## 3. 타입 추론 & Narrowing

- 타입 추론(Type Inference)이란 무엇인가요?
- 타입 추론이 가능한데도 타입을 명시해야 하는 경우는 언제인가요?
- Union Type이란 무엇인가요?
- Narrowing이란 무엇인가요?
- typeof / instanceof / in은 어떤 상황에서 쓰이나요?
- never 타입은 무엇이며 언제 사용하나요?
- exhaustive check는 왜 필요한가요?

## 4. 제네릭 & 유틸리티 타입

- 제네릭(Generic)이란 무엇인가요?
- 제네릭을 쓰는 이유는 무엇인가요?
- 제네릭이 없으면 어떤 문제가 생기나요?
- Partial과 Required는 언제 사용하나요?
- Pick과 Omit의 차이는 무엇인가요?
- Readonly는 언제 유용한가요?

## 5. 객체 & 클래스 타입

- readonly는 무엇이며 왜 사용하나요?
- readonly와 const의 차이는 무엇인가요?
- Optional Property(?)는 언제 사용하나요?
- public / private / protected의 차이는 무엇인가요?
- protected는 private과 어떤 점이 다른가요?
- 접근 제어자를 쓰는 이유는 무엇인가요?

## 6. extends vs implements

- extends와 implements의 차이는 무엇인가요?
- 클래스에서 extends는 무엇을 의미하나요?
- interface에서 extends는 어떤 의미인가요?
- implements는 무엇을 강제하나요?
- implements는 여러 개를 동시에 할 수 있나요?
- 상속(extends)과 구현(implements)의 목적 차이는 무엇인가요?

## 7. 타입 안전성

- 타입 단언(Type Assertion)이란 무엇인가요?
- 타입 단언을 남용하면 어떤 문제가 생기나요?
- any와 unknown의 차이는 무엇인가요?
- unknown이 any보다 안전한 이유는 무엇인가요?
- unknown 값을 사용하려면 왜 타입 체크가 필요한가요?
