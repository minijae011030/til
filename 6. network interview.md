# 네트워크 면접 질문 리스트

## 1. 웹 통신 개요

- 웹 프로토콜이란 무엇인가요?
- ⭐️ 클라이언트와 서버는 어떤 방식으로 통신하나요?
- ⭐️ 요청(Request)과 응답(Response)의 구조를 설명해보세요.

## 2. HTTP 기본 개념

- ⭐️ HTTP란 무엇인가요?
- ⭐️ HTTP의 무상태성(stateless)이란 무엇인가요?
- ⭐️ HTTP의 무상태성이 가지는 장점과 단점은 무엇인가요?
- ⭐️ HTTP에서 상태를 유지해야 할 경우 어떻게 해결하나요?

## 3. HTTP vs HTTPS

- ⭐️ HTTP와 HTTPS의 차이는 무엇인가요?
- ⭐️ HTTPS는 왜 더 안전한가요?
- HTTPS는 어떤 공격을 방지할 수 있나요?
- HTTP를 HTTPS로 바꾸면 성능에 영향이 있나요?

## 4. HTTPS 동작 원리

- ⭐️ HTTPS 통신 과정을 간단히 설명해보세요.
- TLS Handshake란 무엇인가요?
- ⭐️ 대칭키와 비대칭키는 각각 언제 사용되나요?
- ⭐️ 중간자 공격(Man-in-the-Middle Attack)이란 무엇인가요?

## 5. TCP / UDP

- ⭐️ TCP와 UDP의 차이는 무엇인가요?
- ⭐️ 웹 통신은 왜 TCP를 사용하나요?
- UDP가 사용되는 대표적인 사례에는 무엇이 있나요?

## 6. TCP 신뢰성 보장

- TCP는 어떻게 신뢰성을 보장하나요?
- TCP에서 재전송은 언제 발생하나요?
- 흐름 제어와 혼잡 제어의 차이는 무엇인가요?

## 7. TCP 연결 과정

- ⭐️ TCP 3-Way Handshake란 무엇인가요?
- TCP 4-Way Handshake란 무엇인가요?
- TIME_WAIT 상태는 왜 필요한가요?

## 8. REST API

- ⭐️ REST API란 무엇인가요?
- ⭐️ REST API가 stateless해야 하는 이유는 무엇인가요?
- REST API에서 자원(Resource)이란 무엇인가요?

## 9. HTTP 메서드

- ⭐️ GET과 POST의 차이는 무엇인가요?
- PUT과 PATCH의 차이는 무엇인가요?
- ⭐️ 멱등성(Idempotency)이란 무엇인가요?

## 10. HTTP Status Code

- ⭐️ HTTP 상태 코드는 왜 필요한가요?
- ⭐️ 2xx, 4xx, 5xx 상태 코드의 차이는 무엇인가요?
- ⭐️ 400, 401, 403의 차이는 무엇인가요?
- 500 에러는 언제 발생하나요?
